  ROLE GLOSSARY
  ─────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────
  ADMIN    : System administrator. Has full access to all features.
  MAKER    : Staff who submits data change requests for approval.
  CHECKER  : Supervisor who reviews and approves or rejects Maker submissions.
             Note: A Checker may not approve a request they submitted themselves.
  VIEWER   : Read-only user. May view data only. Cannot create, edit, or approve anything.

  EXECUTION NOTES

  Run Order
  ─────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────
  Execute sections in order: 1 → 2 → 3 → 4 → 5 → 6.
  Section 3 depends on pending records created during Section 2.
  Section 6 role tests require separate login sessions per role.

  Prerequisite Users
  ─────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────
  Seeded by default (available on fresh install):
    Username : admin  |  Password : admin123  |  Role : ADMIN

  For role-specific tests (Sections 3 and 6), first create and approve:
    A MAKER user   — for testing Maker-only actions
    A CHECKER user — for testing Checker-only actions
    A VIEWER user  — for testing Viewer restrictions

  Data Isolation
  ─────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────
  Tests in Sections 2 and 3 create records in the database.
  Re-running tests requires either a fresh database or unique usernames
  (e.g., append a timestamp: maker_20260227).
